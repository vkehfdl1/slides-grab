## Slide Specifications

- **Size**: 720pt Ă— 405pt (16:9)
- **Font**: Pretendard (CDN: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.min.css`)
- **Text tags**: Only `<p>`, `<h1>`-`<h6>`, `<ul>`, `<ol>`, `<li>` â€” never put text directly in `<div>` or `<span>`
- **Colors in CSS**: Use `#` prefix (`#FF0000`)
- **Colors in PptxGenJS**: No `#` prefix (`FF0000`)
- **No CSS gradients**: Rasterize to PNG with Sharp instead

## Rules for Agents

1. **Do not modify SKILL.md philosophy sections** â€” design principles, rules, and constraints must be preserved when editing SKILL.md files.
2. **All new scripts go in `scripts/`** unless they are skill-specific (then `skills/<name>/scripts/`).
3. **Use ES modules** â€” `import/export` syntax, not `require()`. Exception: `.cjs` files for CommonJS compat.
4. **JavaScript, not TypeScript** â€” Use JSDoc for type hints if needed.
5. **Test scripts by running them** â€” Use `node scripts/<name>.js` to verify they work.
6. **Playwright browser reuse** â€” When processing multiple slides, keep the browser instance open and reuse it across slides for performance.
7. **Environment variables for API keys** â€” `GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`. Never hardcode keys.
8. **CSS-safe HTML** â€” All text in semantic tags (`<p>`, `<h1>`-`<h6>`, `<ul>`, `<ol>`), backgrounds only on `<div>` elements.
